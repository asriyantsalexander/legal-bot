# 🚀 Запуск бота на сервере Railway — пошаговая инструкция

Всё делается через браузер. Командная строка не нужна.
Время на весь процесс: **~30 минут**.

---

## 📦 Что у вас должно быть в папке

```
юрбот/
├── bot.py
├── requirements.txt
├── Dockerfile
└── railway.toml
```

Все четыре файла — скачаны из чата.

---

## 🔑 ШАГ 1 — Получить Telegram-токен (5 минут)

1. Откройте Telegram на телефоне или компьютере.
2. В строке поиска найдите **@BotFather** (синяя галочка — официальный).
3. Нажмите **Start** (или /start).
4. Отправьте команду:
   ```
   /newbot
   ```
5. BotFather спросит **имя бота** — введите любое, например:
   ```
   Юридический редактор
   ```
6. Затем спросит **username** — должен заканчиваться на `bot`:
   ```
   legal_editor_ivanov_bot
   ```
   (если занято — придумайте другой, добавьте цифры)

7. BotFather пришлёт сообщение с токеном вида:
   ```
   7412345678:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   📋 **Скопируйте этот токен** — он понадобится на Шаге 4.

## 🔑 ШАГ 2 — Получить ключ Anthropic (10 минут)

### 2а. Регистрация
1. Перейдите на сайт: **https://console.anthropic.com**
2. Нажмите **Sign Up**.
3. Зарегистрируйтесь через Google или email.
4. Подтвердите email.

### 2б. Создание ключа
1. После входа нажмите в меню слева: **API Keys**.
2. Нажмите кнопку **Create Key**.
3. Введите любое название, например: `юрбот`.
4. Нажмите **Create Key**.
5. Ключ вида `sk-ant-api03-xxxxx...` появится **один раз**.
   📋 **Сразу скопируйте его** — потом он не покажется!

### 2в. Пополнить баланс
1. В меню слева нажмите **Billing**.
2. Нажмите **Add credit card** → введите данные карты.
3. Нажмите **Add funds** → выберите сумму **$5** (минимум).
4. Подтвердите оплату.

> 💡 $5 хватит примерно на **150–500 проверок** документов.

## 🌐 ШАГ 3 — Зарегистрироваться на Railway (3 минуты)

1. Перейдите на сайт: **https://railway.app**
2. Нажмите **Login** → выберите **Login with GitHub**.
3. Если GitHub нет — нажмите **Login with Email** и зарегистрируйтесь.
4. После входа вас перебросит в личный кабинет.

> 💡 Railway даёт **$5 в месяц бесплатно** — этого хватит для бота.
> Бот потребляет мало ресурсов — вероятно, платить не придётся вовсе.

## 🚢 ШАГ 4 — Загрузить бота на Railway (5 минут)

### 4а. Создать новый проект
1. В личном кабинете Railway нажмите кнопку **New Project**.
2. Выберите **Deploy from GitHub repo** → авторизуйте GitHub, если спросит.

   **Если не хотите использовать GitHub** — выберите **Empty Project**,
   затем нажмите **+ New** → **GitHub Repo** всё равно понадобится.

   > ✅ **Самый простой способ без GitHub:**
   > Выберите **New Project** → **Deploy from GitHub repo** →
   > нажмите **Configure GitHub App** → создайте новый репозиторий прямо там.

### 4б. Загрузить файлы через GitHub (рекомендуется)

**Если у вас нет GitHub:**
1. Зайдите на **https://github.com** → нажмите **Sign up**.
2. Зарегистрируйтесь (бесплатно).
3. Нажмите **+** → **New repository**.
4. Название: `legal-bot` → нажмите **Create repository**.
5. На странице репозитория нажмите **uploading an existing file**.
6. Перетащите все 4 файла (`bot.py`, `requirements.txt`, `Dockerfile`, `railway.toml`).
7. Нажмите **Commit changes**.

**Вернитесь на Railway:**
1. **New Project** → **Deploy from GitHub repo**.
2. Выберите ваш репозиторий `legal-bot`.
3. Нажмите **Deploy Now**.

## ⚙️ ШАГ 5 — Добавить секретные ключи (3 минуты)

После создания проекта на Railway:

1. Нажмите на ваш сервис (квадратик с именем проекта).
2. Перейдите во вкладку **Variables**.
3. Нажмите **New Variable** и добавьте первую переменную:
   - **Name:** `TELEGRAM_TOKEN`
   - **Value:** вставьте токен от BotFather (из Шага 1)
   - Нажмите ✓ (галочку)

4. Нажмите **New Variable** снова и добавьте вторую:
   - **Name:** `ANTHROPIC_API_KEY`
   - **Value:** вставьте ключ Anthropic (из Шага 2)
   - Нажмите ✓

5. Railway автоматически перезапустит бота с новыми ключами.

---

## ✅ ШАГ 6 — Проверить, что бот работает

1. Во вкладке **Deployments** на Railway должен быть статус **Active** (зелёный).
2. Нажмите на деплой → откройте вкладку **Logs**.
3. Найдите строку:
   ```
   Бот запущен. Ожидание сообщений…
   ```
4. Откройте Telegram → найдите вашего бота по username.
5. Отправьте `/start` → бот должен ответить приветствием.
6. Отправьте любой DOCX или PDF файл → получите результат проверки.

🎉 **Бот работает 24/7 на сервере!**

## ❓ Частые вопросы

**Бот не отвечает?**
→ Проверьте Logs на Railway — там будет причина ошибки.
→ Чаще всего: неверно скопированы TELEGRAM_TOKEN или ANTHROPIC_API_KEY.

**Статус «Failed» вместо «Active»?**
→ Нажмите на деплой → Logs → найдите красную строку с ошибкой.
→ Скорее всего, один из ключей пустой или неверный.

**Как обновить бота в будущем?**
→ Отредактируйте файл `bot.py` на GitHub → Railway сам пересоберёт и перезапустит.

**Как остановить бота?**
→ На Railway: Settings → Danger Zone → Delete Service.
